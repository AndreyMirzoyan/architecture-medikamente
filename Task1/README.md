# architecture-medikamente

## Аудит мер по обеспечению безопасности данных

1. Отсутствие классификации обрабатываемых данных.

2. Невыполнение требований законодательства. Компания является оператором конфиденциальных данных, при этом не выполняет требования законодательства
    
3. Отсутствие мер по защите конфиденциальных данных. По этой причине становится возможной утечка персональных данных.

4. Отсутствие мер по авторизации доступа к данным. Все сотрудники имеют доступ ко всем персональным и медицинским данным даже при отсутствии необходимости при вполнении рабочих обязанностей.

5. Отсутствие инструментов аудита действий с данными, мониторинга и логгирования. 

6. Отсутствие резервного копирования данных.


## Предложения по улучшению процессов обработки данных

1. **Первым делом необходимо корректно классифицировать обрабатываемые данные**

Среди обрабатываемых данных присутствуют:
- Персональные данные клиентов
- Персональные данные сотрудников
- Медицинские данные клиентов
- Финансовые данные

2. **Для каждого типа данных необходимо предусмотреть защиту:**

| Вид данных | Способы защиты |
|-|-|
| Персональные данные клиентов | шифрование для хранения и клиент-серверной передачи, обезличивание при передаче в сторонние системы |
| Персональные данные сотрудников | шифрование для хранения |
| Медицинские данные клиентов | шифрование для хранения и клиент-серверной передачи, обфускация и обезличивание при передаче и отсутствии потребности в оригинальных данных |
| Финансовые данные | шифрование для хранения |


3. **Необходимо тегировать все данные:**

По типу данных

- PII - персональные данные
- PHI - медицинские данные (предоставленные клиентом данные и результаты анализов)
- Confidential - конфиденциальные данные

По источнику данных

- Doctor - врач
- Admin - администратор 
- Laboratory - медицинская лаборатория
- Cassier - кассир
- Bookkeeper - бухгалтер
- Warehouse keeper - сотрудник склада

4. **Необходимо осуществлять авторизацию пользователей при попытке доступа к данным**

    При авторизации можно учитывать роли клиентов и сотрудников, а так же выработанные теги

5. **Инструменты**

- Active Directory - для авторизации
- Для шифрования: инструменты Outlook для шифрования эл. писем, TrueCrypt - шифрование файлов "на лету", TLS - при передаче
- Symantec DLP - для предотвращения утечек и аудита безопасности

6. **Далее** 

- Формулирование процессов обработки
- Исключение Excel в пользу защищенной БД (например, с использованием Сфера)
- Исключение прямого доступа к данным, только через порталы или 1С
- Автоматизация процессов обработки
- Глубокое внедрение инструментов аудита
